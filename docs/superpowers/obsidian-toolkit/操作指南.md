## 🚀 执行步骤

### 步骤 1: 确认目标库

1. 检查用户 Obsidian 中实际存在的库
2. 确认用户希望部署到哪个库（如"读书笔记"）
3. 如果库不存在，需要创建或选择现有库

```bash
# 检查现有库
ls "/Users/foldcat/Library/Mobile Documents/iCloud~md~obsidian/"
ls "/Users/foldcat/Library/Mobile Documents/com~apple~CloudDocs/"
```

### 步骤 2: 复制工具包

将工具包复制到目标 Obsidian 库的根目录：

```bash
# 示例：复制到 Documents 库
TARGET="/Users/foldcat/Library/Mobile Documents/iCloud~md~obsidian/Documents"
cp -r "/Users/foldcat/Documents/trae_projects/Novel Understanding/docs/superpowers/obsidian-toolkit" "$TARGET/"
```

### 步骤 3: 安装必需插件

在 Obsidian 中安装以下插件：

| 插件名称 | 必需 | 安装方式 |
|---------|------|---------|
| **Dataview** | ✅ | 社区插件市场 |
| **Templater** | ✅ | 社区插件市场 |
| **Timeline** | ✅ | 社区插件市场 |

**安装步骤**:
1. 打开 Obsidian → 设置 → 社区插件
2. 关闭安全模式（如果提示）
3. 浏览社区插件市场
4. 搜索并安装上述插件
5. 启用所有已安装插件

### 步骤 4: 配置 Dataview

1. 设置 → Dataview
2. 启用选项：
   - ✅ Enable JavaScript queries
   - ✅ Enable inline JavaScript
3. 索引位置: `/`

### 步骤 5: 配置 Templater

1. 设置 → Templater
2. 配置选项：
   - **Template folder**: `obsidian-toolkit/01_templates`
   - ✅ Trigger on new file creation
   - ✅ Trigger on file modification

### 步骤 6: 创建必要文件夹

在 Obsidian 库根目录创建以下文件夹：

```
库根目录/
├── characters/        # 角色笔记
├── chapters/         # 章节笔记
├── factions/         # 阵营笔记
├── plotlines/        # 情节线笔记
├── scripts/          # 脚本（复制 02_scripts 内容）
├── visualization/    # 可视化（复制 03_visualization 内容）
└── templates/        # 模板（复制 01_templates 内容）
```

### 步骤 7: 复制模板到正确位置

```bash
# 复制模板到 templates 文件夹
cp -r "obsidian-toolkit/01_templates/"* "templates/"

# 复制脚本到 scripts 文件夹
cp -r "obsidian-toolkit/02_scripts/"* "scripts/"

# 复制可视化到 visualization 文件夹
cp -r "obsidian-toolkit/03_visualization/"* "visualization/"
```

### 步骤 8: 测试模板

1. 按 `Ctrl/Cmd + P` 打开命令面板
2. 运行 `Templater: Create new note from template`
3. 选择 `角色模板`
4. 创建测试角色笔记
5. 验证 YAML 元数据是否正确生成

### 步骤 9: 测试脚本

1. 打开 `scripts/关系图生成器.md`
2. 确保有角色笔记在 `characters/` 文件夹
3. 运行 DataviewJS 代码块
4. 验证是否生成 Mermaid 代码

### 步骤 10: 验证可视化

1. 打开 `visualization/01_全局关系图.md`
2. 确认 Mermaid 图正常渲染
3. 打开 `visualization/02_全局时间线.md`
4. 确认 Timeline 正常显示

## ⚠️ 常见问题处理

### 问题 1: 插件安装失败
- 检查网络连接
- 尝试重启 Obsidian
- 检查社区插件市场是否可用

### 问题 2: DataviewJS 不执行
- 确认 Enable JavaScript queries 已启用
- 检查代码块是否使用 `dataviewjs` 标记
- 检查文件夹路径是否正确

### 问题 3: Templater 模板不工作
- 确认 Template folder 路径正确
- 检查模板语法是否正确（使用 `<% %>`）
- 重启 Obsidian

### 问题 4: YAML 字段问题
- 确保使用 camelCase（如 `timelineEvents`）
- 检查缩进是否正确（2个空格）
- 确保没有特殊字符
